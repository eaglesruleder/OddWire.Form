# Doc - Form Controls

**Scope:** Subsystem  
**Domain:** oddform  
**Status:** Active

---

## Summary

**Goal:** Provide reusable dynamic-form controls that render from resolved JSON/control props and emit a consistent `onChange(value, param, subkey?)` contract.

**Lifecycle:**
1. `ControlItem` resolves definition + instance overlay and picks a control by `type`.
2. Leaf controls render the current props; most use `ControlBase` for label, hidden, readonly, and row layout.
3. User edits call `props.onChange?.(value, props.param)`.
4. Layout controls keep local UI state and render child controls through nested `ControlList`.
5. Loopers render each row value as a child sparse instance over the same template controls and write the replaced row array back to the looper param.

**Rules:**
- Controls do not own saved values.
- `param` is the DOM id/name where relevant and the update key for `onChange`.
- Leaves that use `ControlBase` inherit hidden gating and readonly replacement.
- `ControlText` and `ControlCheckbox` do not use `ControlBase`; each handles its own hidden gate.
- `readonly: true` displays the value as static text; `readonly: fn` renders the function result in place of the field.
- Radio/dropdown options are either static `controls` or lookup-resolved upstream in `ControlItem`.
- Layout controls store only UI state, never instance values.
- Looper stores repeated row instances in its parent `value`; child params are row-scoped, not root-scoped.

**State:** Parent/runtime owns values. Local UI state exists only for collapsible expanded state, active tabs, popup open state, and editor-specific settings UI.

## Shared Types

### CoreControlProps

```ts
type CoreControlProps<TValue> = {
    param: string;
    label?: string;
    value?: TValue;
    hidden?: boolean;
    stacked?: boolean;
    disabled?: boolean;
    placeholder?: string;
    className?: string;
    cellClassName?: string;
    rows?: number;
    readonly?: boolean | ((props: CoreControlProps<unknown>) => ReactNode);
    onChange?: (value: TValue, key: string, subkey?: string) => void;
    };
```

`cellClassName` is JSON-authorable and lands on the `ControlBase` row for grid spans. `className` is a general style hook, mostly runtime-authored.

### ControlDef

The union covers:
- `label`
- `text`
- `textarea`
- `checkbox`
- `image`
- `radio`
- `dropdown`
- `collapsible`
- `tab`
- `popup`
- `looper`

Radio/dropdown controls can carry `controls?: ControlOption[]` and `dbOptions?: DbOptions`. Layout controls carry child `controls: ControlDef[]`.
Looper controls carry `controls: ControlDef[]`, `value?: LooperRowInstance[]`, and optional `addRows`.
Any control can carry `pdf?: Record<string, PdfBox[]>`, where the string key is a page index and each box has `{ x, y, w?, h? }` (`x, y` is the text's bottom-left anchor). A control can also carry `export?: { pdf?: { fontSize? } }` to override `settings.export.pdf.fontSize` for its own slots.

### DbOptions

```ts
type DbOptions = string | {
    table: string;
    valueParam: string;
    labelParam?: string;
    filter?: string | { formParam: string; tableParam: string };
    joinOptions?: boolean;
    fill?: boolean;
    };
```

Resolution happens in `ControlItem` / `lookup`, not inside the leaf components.

## Leaf Controls

### ControlBase

Shared label+field wrapper.

```text
hidden -> null
rowClass -> flex + column/items-center + gap + mb-3 + stack-sm + className + cellClassName
readonly -> static replacement field
else -> children
```

Renders optional label with `.control-label` and field with `.fill`.

### ControlText

Static display block. Does not use `ControlBase`.

```text
hidden -> null
labelFor target hidden -> null
label -> .separator
value -> .control-static
className -> wrapper style hook, e.g. bubble
```

### ControlTextField

Single-line Bootstrap text input.

```text
inputMode = keyboardType ?? keyboardForValueType[valueType ?? "text"]
value = props.value ?? ""
onChange -> raw string
```

`valueType` values: `text`, `int`, `decimal`, `email`, `phone`.

### ControlTextArea

Bootstrap textarea. Forces `stacked` when spreading props into `ControlBase`.

```text
rows = props.rows ?? 3
```

### ControlCheckbox

Own row layout. Label fills left; checkbox sits right.

```text
checked = props.value ?? false
onChange -> e.target.checked
```

### ControlImage

Display-only image control through `ControlBase` with `stacked`.

```text
value present -> <img src=value>
value absent -> muted placeholder or "No image"
```

There is no upload/capture path yet.

### ControlRadio

Inline Bootstrap radio group.

```text
no options or disabled -> muted placeholder / "No Options"
else -> one Form.Check per option
checked = props.value === option.value
onChange -> option.value
```

Options may be static or lookup-resolved upstream.

### ControlDropdown

Searchable combobox backed by a text input and option menu.

```text
selected option -> query label when closed
focus/click -> open menu
typing -> filter options by label
arrow keys -> move highlight
Enter -> choose highlighted option
choose option -> set query label, close menu, onChange(option.value)
outside pointer down -> close and restore selected label
```

Lookup behaviour is resolved before the dropdown receives props. `dbOptions.fill` is also handled in `ControlItem`, where selected row columns are batched into the instance through `setValues`.

### ControlButton

Small Bootstrap action wrapper:

```text
label + onClick + disabled + variant + size + className
```

Used by popup openers and settings editors.

### ControlError

Error display through `ControlBase`, used for unknown control types and form/instance load errors.

## Layout Controls

Layout controls receive child `controls`, an `InstanceEntity`, and an `onChange`. Collapsible/tab/popup render nested `ControlList`s over the same flat instance. Looper renders each row through a child instance and writes the row array to its own parent value.

`depth` is threaded through nested layouts so sticky bars stack:

```text
stickyTop(depth) = 60 + depth * 48
```

### ControlCollapsible

```text
hidden -> null
expanded = local useState(false)
header button toggles expanded
subtitle renders at header right when supplied
expanded -> ControlList(children, same instance, same onChange, depth + 1)
```

Header is sticky under the strip masthead.

### ControlTab

Takes derived sections:

```ts
type TabSection = {
    param: string;
    label: string;
    controls?: ControlDef[];
    content?: ReactNode;
    disabled?: boolean;
    notice?: string;
    };
```

JSON tabs use `controls`. Programmatic callers, such as DB Manager, can pass rendered `content` and `disabled`.

```text
first enabled section -> default active
defaultParam -> preferred active when valid
disabled section -> cannot activate
notice -> ControlError above body
content present -> render content
else -> render nested ControlList when instance/onChange are present
```

Modes:
- inline: sticky top bar, wraps, child depth increases
- root `pageLayout`: fixed bottom bar, even-width tabs, child depth resets to 0

### ControlPopup

Button opener plus Bootstrap modal. Uses `StripLayout` inside the modal for a title/header.

```text
hidden -> null
ControlButton opens modal
left header button closes modal
content present -> render content
else -> render child ControlList when instance/onChange are present
```

Programmatic callers may pass `content`, `right`, `triggerVariant`, and `triggerSize`. JSON popups pass child `controls`.

### ControlLooper

Repeats a template control list over row-scoped sparse instances. The looper itself adds no row chrome; row UX belongs in the template, e.g. a nested collapsible with `label: "{name}"`.

```text
hidden -> null
rows = Array.isArray(value) ? value : []
each row -> child InstanceEntity(row)
row onChange -> child.setValue(key, subkey, value), then replace row in parent value
addRows -> ControlButton("+ Add Row")
Add Row -> onChange([...rows, { controls: [] }], param)
```

`addRows` defaults false; JSON forms opt in when the user should be able to create rows manually.

`dbOptions.fill` works inside a row because `ControlItem` mutates the child entity before the row `onChange` replaces the parent row array.

## Styling

Global keyword CSS:
- `theme.css` - tokens
- `layout.css` - strip shell and root sizing
- `alignment.css` - flex helpers and 12-column grid helpers
- `text.css` - text helpers
- `controls.css` - label/static/separator/bubble/control stack styles

Control-list layout:
- `.control-grid` wraps every `ControlList`
- children are full-width by default
- `cellClassName` can apply `.col-N`, `.sm-col-N`, or `.lg-col-N`

Colocated layout CSS:
- `layout/layoutControls.css`
- collapsible panel/header/body styles
- inline/root tabset styles
- popup modal sizing/content scrolling

---

## Current Limits

- File/image capture is not implemented; image is display-only.
- Radio/dropdown do not perform validation beyond disabled/missing-option UI.
- Looper rows cannot be removed or reordered yet.
- Dropdown schema editing in DB Manager is simple-type only; option editing is future work.
- Duplicate `param` values can collide in DOM ids and instance identity; runtime compression is last-wins.
