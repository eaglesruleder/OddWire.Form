# Brief — Form Controls

**Scope:** Subsystem
**Domain:** oddform
**Status:** Active

---

## What it is

A library of stateless leaf form controls that render a label+field row through a shared `ControlBase` and report edits through a common `onChange(value, key, subkey?)` contract (each leaf passes its own `param` as the key), plus structural layout controls (collapsible, tab, popup) that render nested control lists over the same flat instance and the same root `onChange`.

---

## Core Loop

```
Leaf:   Receive props → ControlBase lays out label + field → user edits field → onChange(value, props.param) → parent owns the value
Layout: Receive props + instance + onChange → manage local UI state → render nested ControlList over the same instance → child edits call the same root onChange
```

Inputs: **CoreControlProps** (param, label, value, hidden, stacked, onChange) + per-control extras; layout controls additionally take the `instance` and child `controls`
Output: **onChange(value, key, subkey?)** — controls are stateless; the parent holds the value; leaves supply `props.param` as the key. Layout controls hold only local UI state (expanded / active tab / popup open), never an instance value.

---

## Driving Values

| Value | What drives it | What it does |
|---|---|---|
| `hidden` | props | ControlBase renders nothing when true |
| `stacked` | props | Label above (column) vs label-left row |
| `valueType` | ControlTextField | Derives the input's keyboard hint |
| `controls` | Radio/Dropdown config; layout config | Static option list (radio/dropdown) or child control list (collapsible/tab/popup) |
| `variant` | ControlTab (caller-set) | `inline` (sticky-top, wraps) vs `root` (fixed floor bar, split N) |
| `className` | ControlText | Opt-in block style (e.g. `bubble`) |

---

## Class Responsibility

| Class | Owns |
|---|---|
| `ControlBase` | Label+field row layout + hidden gate |
| `ControlText` | Static read-only text (label = separator, value = dump) |
| `ControlTextField` | Single-line input; raw string; keyboard hint |
| `ControlTextArea` | Multi-line input; always stacked |
| `ControlCheckbox` | Boolean; own row (label fills, box right) |
| `ControlRadio` | Static single-select radio group |
| `ControlDropdown` | Static single-select dropdown |
| `ControlButton` | Plain action button (backs the ControlPopup opener) |
| `ControlCollapsible` | Panel-wide collapsible section; local expanded state; nested ControlList |
| `ControlTab` | Tabset over derived sections; local active tab; inline (sticky-top) or root (fixed floor) variant |
| `ControlPopup` | Opener button + react-bootstrap Modal reusing StripLayout as its banner; nested ControlList |
