# Doc — Dynamic Form

**Scope:** Feature
**Domain:** oddform
**Status:** Active

---

## Summary

**Goal:** Render a JSON form definition overlaid by a live instance array, dispatching each control on its resolved `type` and mutating the instance by `param`.

**Lifecycle:**
1. FormPage awaits `getForm(id)` + `getInstance(id)` together, then clears one `loading` flag (splash until both resolve)
2. Missing form → `ControlError` "Form Not Found"; else hand the loaded instance to `InstanceEntity`
3. Pass `form.controls` + the `instance` (the `InstanceEntity`) to `ControlList`
4. ControlList hands each form control plus the instance to a `ControlItem`
5. ControlItem calls `instance.resolve(control)` (merge boundary), switches on resolved `type` (unknown → `ControlError`)
6. User edits → `onChange(value, param, key)` → `instance.setValue(param, key, value)` → `forceUpdate` + `InstanceContext.set(instance, id)` (persist)

**Rules:**
- Type dispatch + merge live only in `ControlItem`; `ControlList` stays type-agnostic
- Unknown `type` (and a missing form) render a `ControlError` box, never crash
- Hidden controls render nothing (each leaf / `ControlBase` gates `hidden`)
- Instance overlays by `param`; instance props override form props; repo compresses duplicate params (last-wins) on load + save
- `controls` stays an array; `setValue` replaces the array (new ref) and `forceUpdate` re-renders
- Contexts are thin data repos; the live instance is owned by `InstanceEntity`, persisted via localforage (IndexedDB)

**State:** `Stored: form (FormDefinition|null) in FormPage, active instance in InstanceEntity — Persisted: instances via localforage on every edit`

## Structure

`FormPage` → `src/form/` · `FormContext`/`InstanceContext` + `FormDefinition`/`FormInstance`/`ControlInstance` (`types.ts`) + `data/forms/*.json` + `data/instances/*.json` + barrel → `src/_context/` · `ControlList` + `ControlItem` + `controls/` → `src/_components/controllist/` (this folder). `ControlDef` lives in `controls/controlTypes.ts`, re-exported from the controllist barrel. Control leaves + their contracts map is `controls.gpt.md` in `controls/`.

## Implementation

### FormContext / InstanceContext
Thin data-repo APIs (value = the `persistent` store, supplied by `ContextsProvider`) — get/set by id, no entity logic:
- `FormContext.getForm(id): Promise<FormDefinition>`
- `InstanceContext.getInstance(id): Promise<FormInstance>` and `set(instance, id)` — the autosave write-back

### persistentStore
`PersistentStore` singleton behind localforage (IndexedDB, localStorage fallback). `initialise()` loads `forms`/`instances`, seeding from bundled JSON on first run only; `getForm`/`getInstance` resolve plain data by id; `set(instance, id)` upserts by id and persists the array. `compressDuplicates` collapses same-`param` controls last-wins on both load (`getInstance`) and save (`set`). Structured/binary values store natively (ready for PDF/image blobs).

### InstanceEntity
The live instance as one thing — a value object that is **also** a React class component (merges the old `FormActiveInstance` + render-prop). The overlay lives in a plain `instance` field (not React state), so the value-object methods work mounted or detached:
- `get(param)` — last-wins match; dev `console.warn` on duplicate `param`
- `resolve(control)` — `{ ...control, ...get(control.param) }`, cast once at the merge boundary
- `setValue(param, key, value)` — merge the changed key into that param's control; points the field at a fresh object (void)
- `static from(data)` — object mode: a detached entity around plain data, no React
- Mounted: takes the loaded `instance` as a prop (no async in the entity); `onChange(value, param, key='value')` → `setValue` + `InstanceContext.set(instance, id)` + `forceUpdate`; `render` → `children(this, onChange)`

### FormPage
Awaits `getForm` + `getInstance` together behind one `loading` flag. Missing form → `ControlError` "Form Not Found"; else renders `StripLayout > Form > InstanceEntity` (seeded with the loaded instance), whose render prop feeds `form.controls` + the entity + `onChange` into `ControlList`.

### ControlList
Type-agnostic iteration. Hands each form `control` plus the whole `instance` to a `ControlItem`. No merge, no type knowledge.

### ControlItem
The resolve + dispatch boundary. `instance.resolve(control)` merges form + instance, then `switch(resolved.type)` spreads into the matching leaf (interactive leaves also get `onChange`); `default` → `ControlError` reading `Unknown control type: {type}`.

### ControlError
The error box (`ControlBase label="Error" className="error" labelClassName="bold"`). Used by `ControlItem` (unknown type) and `FormPage` (form not found).

### StripLayout
Headered tablet-portrait shell — masthead title + padded `main`. Strip width and borders live on `#root` (layout.css); this only supplies the header band and body slot.
