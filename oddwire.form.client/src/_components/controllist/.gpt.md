# Doc — Dynamic Form

**Scope:** Feature
**Domain:** oddform
**Status:** Active

---

## Summary

**Goal:** Select a form + instance from local stores, render the JSON definition overlaid by a live instance array dispatched on resolved `type`, and persist edits by `param` after an explicit first save.

**Lifecycle:**
1. `ContextsProvider` initialises `formStore` → `instanceStore` → `lookupStore` behind a splash (forms first — instance display projection reads the form index)
2. `LandingPage` `list()`s forms → expand → `list(formId)`s instances (or New) → link to `/form/:formId/:instanceId?`
3. `FormPage` awaits `getForm(formId)` + resolves the instance behind one `loading` flag
4. Absent `instanceId` → an unsaved in-memory `InstanceEntity` (`autosaving=false`); present → `getInstance` (`autosaving=true`)
5. `ControlList` hands each form control + the instance to `ControlItem`; `ControlItem` calls `instance.resolve(control)`, switches on resolved `type` (unknown → `ControlError`)
6. Edit → `onChange(value, key, subkey)` → `entity.setValue(key, subkey, value)` → `bumpRender`; persists via `save` only while `autosaving`
7. Header save icon → `save(entity.instance)` (mints/keeps GUID, stamps `dateModified`, re-projects index) → `autosaving=true`; a new instance also `navigate`s to its GUID URL

**Rules:**
- Type dispatch + merge live only in `ControlItem`; `ControlList` stays type-agnostic
- Unknown `type` (and a missing form/instance) render a `ControlError` box, never crash
- A new instance persists nothing until the first hard save — no save-on-mount (avoids double-save under StrictMode / effect re-run)
- Instance overlays by `param`; instance props override form props; the store compresses duplicate params last-wins on load + save
- Instance `dateModified` + GUID identity are stamped by the store / entity, never by callers; form `dateModified` is persisted as provided (forms are published)
- `list()` reads only the derived index tables; bodies load by id
- `display` is projected from the form's `displayParam` at instance-save time; a form `saveForm` that changes `displayParam` triggers `instanceStore.reindexForm` to refresh existing instances
- radio/dropdown `dbOptions` resolve options from the aggregated lookup db (provided at `FormPage` via `DbContext`), save `valueParam` / display `labelParam`; missing table or zero (post-filter) rows → disabled + "No Options" placeholder; the control saves the key, never the label

**State:** `Stored: form + InstanceEntity + autosaving in FormPage — Persisted: form/instance bodies (GUID keys) + index tables via localforage; new instance only after first Save`

## Implementation

### FormContext / FormStore
Context value **is** the `formStore` singleton (localforage `storeName: 'forms'`). GUID body keys + one `index` (derived `FormIndexEntry[]`).
```
initialise()
  #region load index; seed each form through saveForm when empty
getForm(id)          — body by GUID key, undefined on miss
list()               — the in-memory index
getDisplayParam(id)  — index lookup → displayParam ?? []  (instance FK read)
saveForm(form)
  #region form.formId ??= uuid   // identity on the object; dateModified persisted as provided (forms are published, not stamped here)
  #region displayParamChanged = prior index entry && !sameParams(prior.displayParam, form.displayParam)
  #region setItem(formId, form); refreshIndex(form)
  #region if(displayParamChanged && instanceStore.list(formId).length) instanceStore.reindexForm(formId)
refreshIndex(form)   — project {formId,label,version,displayParam,dateModified}; upsert; persist
```

### InstanceContext / InstanceStore
Context value **is** the `instanceStore` singleton (`storeName: 'instances'`). GUID body keys + one `index` (`InstanceIndexEntry[]`).
```
initialise()   — load index; seed each instance through save when empty
getInstance(id)— body by GUID; compressDuplicates; undefined on miss
list(formId)   — index filtered by formId
reindexForm(formId) — reload each affected body, refreshIndex it (re-projects display off the form's new displayParam); called by FormStore on displayParam change
save(instance)
  #region instance.instanceId ??= uuid; instance.dateModified = now
  #region body = compressDuplicates(instance); setItem(instanceId, body); refreshIndex(body)
refreshIndex(instance) — {instanceId,formId,dateModified,display: projectDisplay}; upsert; persist
projectDisplay(instance) — for each formStore.getDisplayParam(formId): instance value by param
```
`compressDuplicates` collapses same-`param` controls last-wins on load + save. `upsert` (shared `storeUtils`) replaces-or-appends by match.

### LookupContext / LookupStore
Context value **is** the `lookupStore` singleton (`storeName: 'lookup'`, one `db` key). Whole database held in memory for synchronous option resolution; persisted on every mutation. Storage is `Record<scope, Record<tableName, LookupTable>>` where `GLOBAL_SCOPE = '_global'` holds shared tables (form-local layer deferred).
```
initialise()   — load db; seed each table through saveTable('_global', …) when empty
get(formId)    — merge db[_global] then overlay db[formId] → Record<tableName, LookupTable>  (whole-table override; local layer empty for now)
listTables(scope) / getTable(scope, name) — sync reads off the in-memory db
saveTable(scope, table)   — stamp lastUpdated; persist
deleteTable(scope, name)  — drop from scope; persist
importRows(scope, name, rows, schema?) — replace rows only; keep existing schema, else inferSchema (a text column per first-row key)
```

### ContextsProvider
Awaits `formStore` → `instanceStore` → `lookupStore` initialise (forms first) behind a splash `StripLayout`; provides all three stores as context values.

### InstanceEntity
Plain value class over the overlay — no React. `static from(instance)` wraps data and mints `instanceId` if absent; `instanceId` getter; `get(param)` last-wins (dev warn on duplicate); `resolve(control)` = `{...control, ...get(param)}`; `setValue(param, subkey, value)` merges the changed subkey into that param's control, pointing the field at a fresh object. `mergeInstances(base, ...overlays)` (module fn) overlays instances by `param` (overlay fields win) — used to project display-only prop overrides (e.g. `{hidden}` / `{type}`) onto a data instance without touching the stored values.

### FormPage
Route params `formId` / `instanceId`. State: `loading`, `form`, `instance`, `autosaving`, `bumpRender`.
```
useEffect [getForm,getInstance,formId,instanceId]
  #region getForm(formId) → setForm
  #region resolveInstance: !instanceId → unsaved fresh entity (autosaving=false); else getInstance → entity + autosaving=!!loaded
onChange(value,key,subkey='value') — setValue; save only while autosaving; bumpRender
onSave() — id = await save(instance); autosaving=true; new (no instanceId) → navigate replace to /form/formId/id
```
Error states via `ControlError`: no form → "Form Not Found"; missing instance → "Instance Not Found". Renders `StripLayout(left ← "/" · right saveIcon 💾 · title form.label) > DbContext.Provider value=getDb(formId) > Form > body`. Save icon disabled + "Autosaving" once `autosaving`.
```
isRootTab = form.controls[0]?.type === 'tab'
  #region isRootTab → ControlTab pageLayout sections=buildRootTabSections(controls); else ControlList
buildRootTabSections(controls) — visible tabs → sections; visible non-tab strays → one trailing '⚠' section, notice 'Unexpected controls in tab layout' (hidden filtered from both)
```

### App / routing
`BrowserRouter` → `Routes`: `/` `LandingPage`, `/form/:formId/:instanceId?` `FormPage`, `/settings` `SettingsPage`, `*` → `/`.

### LandingPage
`listForms()` → form buttons (label + version); toggle expands one form to `listInstances(formId)` → `Link` per instance (`display` join, `"Untitled instance"` fallback, + `dateModified`) plus a New `Link` to `/form/:formId`. Settings gutter link.

### ControlList
Iterates controls, folding each contiguous run of `tab` controls into one inline `ControlTab` tabset; everything else dispatches 1:1 to `ControlItem`. Wraps the items in a `.control-grid` (12-col grid, children full-width by default) so a control's JSON `cellClassName` (`col-N`) opts into columns. Passes the whole `instance` down.
```
while i < controls: control = controls[i]
  #region control.type != 'tab' → ControlItem; i++
  #region collect run of adjacent tabs → sections (skip hidden); if any → ControlTab (inline)
```

### ControlItem
Resolve + dispatch boundary. Reads `DbContext`, then `instance.resolve(control)` merges, then `switch(resolved.type)`: leaves spread `{...resolved}` (+ `onChange` for interactive); radio/dropdown also spread `optionSource(dbOptions, controls, db, instance)`; layout `collapsible`/`popup`/`tab` also get `instance` (they render nested `ControlList`s). A lone `tab` here → a single inline tabset (runs are normally grouped by `ControlList`). `default` → `ControlError` "Unknown control type: {type}".
```
optionSource(dbOptions, staticControls, db, instance)
  #region !dbOptions → {} (control keeps its own controls/disabled/placeholder)
  #region else resolveDbOptions(...) overrides controls/disabled/placeholder
```

### lookup/ (DbContext + resolveDbOptions)
`DbContext` — the aggregated `Record<tableName, LookupTable>` map; `FormPage` builds it via `lookupStore.get(formId)` and provides it around the form body (default `{}`, so `ControlList` used elsewhere needs no provider).
```
resolveDbOptions(dbOptions, db, instance, staticControls) → {controls, disabled, placeholder?}
  #region normalise: string shorthand → {table, valueParam=labelParam=schema[0].param}; filter string → {formParam==tableParam}
  #region !db[table] → {staticControls, disabled, "No Options"}
  #region applyFilter: no filter → all rows; else keep row[tableParam]==instance.get(formParam).value; empty upstream value → [] (dependent list)
  #region map rows → {value:String(row[valueParam]), label:String(row[labelParam ?? valueParam])}; joinOptions → prepend staticControls
  #region 0 options → disabled + "No Options"; else {controls, disabled:false}
```

### ControlError / ControlButton
`ControlError` — error box (`ControlBase label="Error" className="error"`), used by `ControlItem` (unknown type) and `FormPage` (not-found). `ControlButton` — plain action button, currently unused (kept).

### SettingsPage / DbManager (src/settings/)
`SettingsPage` renders a red `Clear Cache` button (`localforage.dropInstance('oddwire.form')` → reload, re-seeds) + `DbManager` in a `StripLayout` (← "/"). `DbManager` (global scope only) is built from the control primitives themselves — owns `selected` table + a `bumpRender` after each persisted mutation:
```
Import (ControlPopup)  — trigger opens a popup; header `right` = ImportFromFileButton (📁, parse JSON rows → prompt table name → importRows), body = FormCatalog (every JSON in data/forms via import.meta.glob; Install/Reinstall → formStore.saveForm)
ControlDropdown      — table picker: options = tables + "+ Add New" (ADD_NEW sentinel → create prompt); a Delete button shows when selected
ControlTab           — two content-tabs "Edit Model" / "Edit Rows" (defaultParam="rows" opens Rows first — avoids accidental schema edits), both disabled when !selectedTable
  Edit Model → DbSchemaEditor — each column is one ColumnRow built from primitives: readonly ControlTextField (param/label) + ControlDropdown (type) + ControlButton actions; ✎ flips the row's `readonly` off to edit (✓ confirm / ✕ cancel), display shows ✎ edit / ✕ delete. The row is a 12-col `.grid` (param `col-4` / label `col-4` / type `col-2` / ✎✕ `col-1` each). Top-right "+ Add" (`.db-add-col`, negative top margin + z-index over the sticky tab bar) prepends a sticky new-column row in edit mode; one draft at a time. Editing a param orphans its row data (not migrated)
  Edit Rows  → DbRowEditor    — pure control primitives: a row-picker ControlDropdown over table rows (keyed by schema[0].param) + "Add New Row", then a ControlList over the schema for the picked row
```
`DbRowEditor` keys on `schema[0].param`. The picked/new row is an `InstanceEntity`; the ControlList's instance is `mergeInstances(entity, keyOverlay)` — existing row hides the key (the dropdown owns it), new row forces the key to an editable `text`. Edits are **live write-through** (`onContextChange`): each change recomputes the row and persists via `onChange` → `saveTable`; a key edit renames (drops the old key, dedupes) rather than duplicating. Schema/row edits rebuild the `LookupTable` and `saveTable('_global', …)`; both editors are `key`ed by table name so state resets on table switch. Edits surface on `FormPage` via normal re-mount on navigation — no cross-store subscription.

### StripLayout
Sticky three-cell masthead: fixed left/right icon gutters (`< 50px`, an icon node + optional route `Link`) around a filling single-line, ellipsised centre title, over padded `main`. `position: sticky; top: 0` pins it on scroll. Strip width/borders on `#root` (layout.css).
