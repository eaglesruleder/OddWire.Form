# Doc - Dynamic Form

**Scope:** Feature  
**Domain:** oddform  
**Status:** Active

---

## Summary

**Goal:** Select a locally stored form + instance, render the form definition over a sparse live instance overlay, resolve lookup-backed controls, and persist edits by `param` after the first explicit save.

**Lifecycle:**
1. `ContextsProvider` initialises `formStore`, then `instanceStore`, then `lookupStore`.
2. `LandingPage` lists installed forms from the form index; URL `FormID` can auto-open one form; expanding a form lists saved instances from the instance index plus a New link.
3. `FormPage` reads `formId` / optional `instanceId` from the route and loads the form + instance behind one loading state.
4. No `instanceId` creates an unsaved `InstanceEntity`; an existing `instanceId` loads and wraps the stored body.
5. `ControlList` iterates the definition controls and groups adjacent tabs; `ControlItem` resolves each control against the instance overlay.
6. Interactive leaves emit `onChange(value, param, subkey?)`; `InstanceEntity` applies the patch and `FormPage` bumps render.
7. A looper renders each row value as a child sparse instance over the looper's template controls, then writes the replaced row array back to its own param.
8. Hard Save writes immediately, mints/keeps the instance GUID, enables autosave, and navigates a new instance to its GUID URL.
9. Later edits schedule debounced persistence through the entity; cleanup flushes pending writes before unmount or instance replacement.

**Rules:**
- Form controls are identified by flat, global `param`; nested layout controls share the same instance scope.
- Looper child controls are scoped to their row instance; the looper param stores the row-instance array in the parent overlay.
- Instance overlay props override definition props.
- Empty/null/undefined `value` removes that param entry from the instance.
- Duplicate instance params compress last-wins on load and save.
- Unknown control `type` renders `ControlError`.
- Missing form/instance route targets render error pages, not crashes.
- Form `dateModified` is preserved as published; instance `dateModified` is stamped by `InstanceStore.save`.
- `displayParam` is a form concern; instance index projections store `{ label, value }` and remain backward-compatible with older raw values.
- `displayParam` may contain `null` separators; the landing page renders them as detail-line breaks.
- `groupParam` URL values open matching groups without filtering away other groups.
- Lookup tables are stored globally this phase; `LookupStore.get(formId)` already supports whole-table form-local override.
- DB-backed radio/dropdown controls store the selected key (`valueParam`) and display the label (`labelParam`).
- `dbOptions.fill` writes non-empty columns from the selected lookup row into matching params before setting the selected key.

**State:** Stored locally in browser `localforage`: form bodies + form index, instance bodies + instance index, lookup database. In React: active form, active `InstanceEntity`, autosave flag, local layout/settings UI state.

## Implementation

### App

`BrowserRouter` routes:
```text
/                         -> LandingPage
/form/:formId/:instanceId? -> FormPage
/settings                 -> SettingsPage
*                         -> Navigate("/")
```

### ContextsProvider

Initialises stores once and shows a `StripLayout` splash until complete.

```text
initialise()
-> await formStore.initialise()
-> await instanceStore.initialise()
-> await lookupStore.initialise()
-> provide FormContext / InstanceContext / LookupContext
```

Forms initialise before instances because instance display projection reads the form index.

### FormContext / FormStore

`localforage` instance: `name: 'oddwire.form'`, `storeName: 'forms'`.

```text
initialise()
-> load index
-> if empty, save seedForms
-> mark initialised

getForm(formId)
-> body by GUID key

list()
-> in-memory index

saveForm(form)
-> form.formId ??= crypto.randomUUID()
-> collect projection labels from the control tree
-> detect projection param/label changes from prior index entry
-> persist body
-> refresh index
-> if projection changed, reindex existing instances for that form
```

Seeded forms currently include test, tab, layout-test, and vehicle lookup forms. Other bundled form JSON files are installable through Form Manager.

### InstanceContext / InstanceStore

`localforage` instance: `name: 'oddwire.form'`, `storeName: 'instances'`.

```text
initialise()
-> load index
-> if empty, save seedInstances
-> mark initialised

getInstance(instanceId)
-> load body
-> compress duplicate params last-wins

list(formId)
-> index filtered by formId

save(instance)
-> instance.instanceId ??= crypto.randomUUID()
-> instance.dateModified = now
-> compress duplicate params
-> persist body
-> refresh index

reindexForm(formId)
-> reload each affected body
-> refresh index from current form display/group/filter/order params
```

Projection reads the current form index, copies matching instance values, and stores each field as `{ label, value }`.

### LookupContext / LookupStore

`localforage` instance: `name: 'oddwire.form'`, `storeName: 'lookup'`, one key: `db`.

```text
initialise()
-> load db
-> if empty, save seed tables into GLOBAL_SCOPE

get(formId)
-> start with db["_global"]
-> overlay db[formId] by whole table name

saveTable(scope, table)
-> stamp lastUpdated
-> replace table in memory
-> persist db

deleteTable(scope, tableName)
-> remove table from scope
-> persist db

importRows(scope, tableName, rows, schema?)
-> new table: schema or inferred schema from first row
-> existing table: keep schema unless explicit schema passed
-> replace rows
-> saveTable
```

The global seed tables are `Model` and `OnLot`.

### InstanceEntity

Store-agnostic live overlay wrapper.

```text
from(instance)
-> clone data
-> ensure instanceId

get(param)
-> find matching instance controls
-> warn when duplicated
-> return last match

resolve(control)
-> return {...definitionControl, ...instanceControl}

setValue(param, subkey, value)
-> applyValue
-> schedulePersist

setValues(record)
-> apply every param as value
-> schedule one persist

setControls(record)
-> apply every param as an overlay patch
-> schedule one persist

applyValue(param, "value", empty)
-> remove that param entry

applyValue(param, subkey, value)
-> merge subkey into existing/new control entry

applyControl(param, patch)
-> merge patch into existing/new control entry

withPersist(fn)
-> install debounced persistence callback

flush()
-> fire pending debounced persist now

cancelPersist()
-> drop pending debounced persist
```

`setValues` is used by `dbOptions.fill` so a selected lookup row can populate several matching params with one persist.

### FormPage

Route state: `formId`, optional `instanceId`. React state: `loading`, `form`, `instance`, `autosaving`.

```text
effect(formId, instanceId)
-> getForm(formId)
-> resolveInstance()
-> loading=false

resolveInstance()
-> no instanceId: create fresh InstanceEntity({formId, controls: []}), autosaving=false
-> same instance already held: autosaving=true, skip reload
-> existing instanceId: load body, wrap entity, autosaving=!!loaded

onChange(value, key, subkey="value")
-> instance.setValue(key, subkey, value)
-> bump render

onSave()
-> cancel pending debounced persist
-> save instance immediately
-> autosaving=true
-> if new route, replace URL with instance id
```

The entity persistence callback reads `autosavingRef` so an unsaved new instance does not write on mount or first edit. Cleanup flushes pending writes before replacing/unmounting the entity.

Rendering:
```text
loading          -> loading strip
missing form     -> ControlError("Form Not Found")
missing instance -> ControlError("Instance Not Found")
root first tab   -> ControlTab pageLayout with root sections
otherwise        -> ControlList
```

The header back link returns to `/?FormID=<formId>` and appends current instance group-param values so the landing page can reopen the matching group.

`buildRootTabSections` converts visible root tabs into fixed footer sections and sends visible non-tab strays into a warning section.

### LandingPage

```text
forms = formStore.list()
-> URL FormID can auto-expand a form
-> render one button per form
-> expanded form renders filter/order controls and grouped instance rows
-> filter URL params preselect filters
-> group URL params open matching groups without hiding others
-> first displayParam = title; remaining display fields = labelled details
-> null displayParam = detail line break
-> New links to /form/:formId
```

### ControlList

```text
while controls remain
-> non-tab: render ControlItem
-> adjacent run of tabs: build sections, skip hidden, render one ControlTab
-> wrap all items in .control-grid
```

The `.control-grid` wrapper makes every child full-width by default; JSON `cellClassName` can opt a control into column spans.

### ControlItem

```text
db = useContext(DbContext)
resolved = instance.resolve(control)
resolved.label = resolveLabel(resolved.label, instance)

switch resolved.type
-> label / text / textarea / checkbox / image
-> radio / dropdown with optionSource and fillOnChange
-> collapsible / popup / tab / looper layout controls
-> default ControlError
```

`optionSource` leaves static props untouched when `dbOptions` is absent. When present, `resolveDbOptions` overrides `controls`, `disabled`, and `placeholder`.

`fillOnChange` applies only to object-shaped `dbOptions` with `fill: true`:
```text
select key
-> find row by valueParam
-> split every other non-empty column into value patches vs control overlay patches
-> value patches call setValues
-> overlay patches call setControls
-> call original onChange for selected key and render
```

Overlay patch columns are objects carrying control props such as `{ hidden: true }` or `{ hidden: false, value: [...] }`. This lets lookup import rows hide or populate layout controls without treating the object as the literal field value.

Looper rows:
```text
value missing/malformed -> []
addRows -> append { controls: [] } to looper value
render row -> child InstanceEntity(row)
row edit -> child entity applies patch
parent onChange -> replace row in looper value
```

### resolveLabel

Interpolates `{param}` tokens in labels from the current instance value. Used by leaf labels, inline tab labels, and root tab labels.

```text
"STR {strSave}" -> "STR +3"
```

### lookup/resolveDbOptions

```text
normalise(dbOptions)
-> string shorthand: table name, value/label = schema[0].param or "id"
-> object config: keep table/value/label/filter/join/fill
-> filter string: formParam == tableParam

resolve
-> missing table: static controls, disabled, "No Options"
-> filter rows by current instance value when filter is present
-> map rows to {value, label}
-> joinOptions: append db controls after static controls
-> zero options: disabled, "No Options"
-> otherwise enabled controls
```

### SettingsPage

`StripLayout` with back link and a clear-all warning button. Clear-all drops the whole `oddwire.form` localforage instance and reloads to reseed.

Renders:
- `DbManager`
- `FormManager`

### FormManager

`FormList` uses `import.meta.glob('../../_context/data/forms/*.json', { eager: true })` to list every bundled form JSON.

```text
not installed -> Install
bundled version newer -> Update
same/older -> Refresh
```

Each action calls `formStore.saveForm(form)` and bumps render.

Monster import now sits in DB Manager and opens a `MonsterSetCatalog`.

### DbManager

Global scope only this phase (`GLOBAL_SCOPE = '_global'`).

```text
table picker
-> options = existing tables + "+ Add New"
-> Add New prompts for table name and creates empty table
-> Delete removes selected table
-> Clear removes rows but keeps schema
-> Import parses JSON row file, prompts target table
```

Import behaviour:
```text
new table
-> infer schema from first row unless schema supplied
-> save all rows

existing table
-> keyParam = schema[0].param ?? "id"
-> no collisions: merge rows
-> collisions: DupResolvePopup chooses overwrite keys
```

Editor tabs:
- `Edit Model` -> `DbSchemaEditor`
- `Edit Rows` -> `DbRowEditor`

`ControlTab` is used programmatically with rendered `content` sections and disabled sections.

### DbSchemaEditor

First-pass schema editor for simple types: `text`, `checkbox`, `dropdown`.

```text
one draft row at a time
-> + Add creates sticky new-column draft
-> edit toggles row readonly off
-> commit trims param/label, rejects empty/clashing param
-> rename does not migrate row data
-> remove deletes schema column
```

Column rows are built from `ControlTextField`, `ControlDropdown`, and `ControlButton`.

### DbRowEditor

Treats a lookup row as a form instance over the table schema.

```text
pick existing row
-> build InstanceEntity from row values
-> hide key param via overlay
-> edits live-update row in place

add new row
-> build draft InstanceEntity
-> make key param editable
-> Save row requires non-empty key
-> colliding key opens DupResolvePopup
```

`mergeInstances(entity.instance, keyOverlay)` lets the row editor alter visibility/editability without mutating stored row values.

### DupResolvePopup

Bootstrap modal with `StripLayout` header. Ticked rows overwrite; unticked rows keep existing. Used by JSON import and add-new-row collisions.

### 5etools Monster Import

`monsterSets` defines remote 5etools bestiary sources and downloads source JSON from the public mirror. It expands supported `_copy` entries before mapping:

```text
load source file
-> cache raw bestiary file
-> build current-file lookup
-> for each _copy: prefer current file by source/name or name
-> else fetch _copy.source file
-> clone resolved base
-> apply supported array _mod operations
-> overlay copied monster fields
```

Supported `_mod` operations are append/prepend/replace/remove/insert array items. Unsupported modes are ignored.

`MonsterSetCatalog` filters source labels, loads one source into a review popup, floats conflicts to the top, lets the user select monsters, sets collection/session metadata, maps selected rows through `mapMonsters`, merges by monster key, and saves the global `Monster` lookup table with `MONSTER_SCHEMA`.

`monsterMapper` flattens a 5etools bestiary entry into string/control-patch columns used by the Monster Reference Card form. It includes actions, bonus/reaction/legendary/mythic actions, spellcasting, traits, variants, damage/condition sections, token image URLs, ability mods, saves, skills, and textarea row hints.

### StripLayout

Three-cell sticky masthead over a padded main area:
- left slot: icon/button/link
- center: ellipsised title
- right slot: icon/button/link

The modal popup controls reuse it as their header.

## Current Limits

- Server persistence/API is not implemented.
- Lookup form-local tables are structurally supported but not surfaced in the DB Manager.
- JSON import only; CSV import is not implemented.
- Schema editor supports only simple column control types.
- Renaming a schema param does not migrate existing row values.
- Looper row removal/reorder, validation/rules, file capture/upload, and server sync are future work.
