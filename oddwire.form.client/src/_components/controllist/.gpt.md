# Doc — Dynamic Form

**Scope:** Feature
**Domain:** oddform
**Status:** Active

---

## Summary

**Goal:** Render a JSON form definition overlaid by a live instance array, dispatching each control on its resolved `type` and mutating the instance by `param`.

**Lifecycle:**
1. FormPage asks `FormContext.getForm(id)` + `InstanceContext.getInstance(id)`
2. While either is unresolved, render a loading shell
3. Pass `form.controls` + the `instance` (a `FormActiveInstance`) to `ControlList`
4. ControlList hands each form control plus the instance to a `ControlItem`
5. ControlItem calls `instance.resolve(control)` (merge boundary), switches on resolved `type` (unknown → `ControlBase` error box)
6. User edits → `onChange(value, param)` → FormPage `instance.patch(param, value)` → re-render

**Rules:**
- State is memory-only — no persistence, lost on refresh
- Type dispatch + merge live only in `ControlItem`; `ControlList` stays type-agnostic
- Unknown `type` renders a `ControlBase` error box, never crashes
- Hidden controls render nothing (each leaf / `ControlBase` gates `hidden`)
- Instance overlays by `param`; instance props override form props; duplicate params resolve last-wins (dev warn)
- Instance stays an array; patches are immutable array replacements

**State:** `Stored: form (FormDefinition | null), instance (FormInstance | null) in FormPage — Persisted: never`

## Structure

`FormPage` → `src/form/` · `FormContext`/`InstanceContext` + `FormDefinition`/`FormInstance`/`ControlInstance` (`types.ts`) + `data/forms/*.json` + `data/instances/*.json` + barrel → `src/_context/` · `ControlList` + `ControlItem` + `controls/` → `src/_components/controllist/` (this folder). `ControlDef` lives in `controls/controlTypes.ts`, re-exported from the controllist barrel. Control leaves + their contracts map is `controls.gpt.md` in `controls/`.

## Implementation

### FormContext
Provides the durable `getForm(id): Promise<FormDefinition>` pathway. Stub body ignores the id and returns the imported `testform.json` (single boundary cast). Later grows the folder-scan + write-to-context; only the body changes.

### InstanceContext
Provides the durable `getInstance(id): Promise<FormActiveInstance>` pathway (mirrors `FormContext`); stub wraps `testinstance.json` in a `FormActiveInstance`.

### FormActiveInstance
The live instance as a behaviour-carrying object — `implements FormInstance` (the sparse overlay) and owns the operations over it, so consumers hold one thing rather than an array plus free helpers:
- `get(param)` — last-wins match; dev `console.warn` on duplicate `param`
- `resolve(control)` — `{ ...control, ...get(control.param) }`, cast once at the merge boundary
- `patch(param, value)` — returns a new `FormActiveInstance` (immutable; a mutated one would not re-render)

### FormPage
Loads definition + instance via context, owns the active instance, holds the single change sink.

- `getForm` / `getInstance` in one effect → `form` / `instance` state; loading shell until both resolve
- `handleChange(value, param)` → `setInstance(prev => prev.patch(param, value))`
- Renders `StripLayout > Form > ControlList` with `form.controls` + the `instance`

### ControlList
Type-agnostic iteration. Hands each form `control` plus the whole `instance` to a `ControlItem`. No merge, no type knowledge.

### ControlItem
The resolve + dispatch boundary. `instance.resolve(control)` merges form + instance, then `switch(resolved.type)` spreads into the matching leaf (interactive leaves also get `onChange`); `default` → bold dark-red `ControlBase label="Error"` box (`className="error" labelClassName="bold"`) reading `Unknown control type: {type}`.

### StripLayout
Headered tablet-portrait shell — masthead title + padded `main`. Strip width and borders live on `#root` (layout.css); this only supplies the header band and body slot.
