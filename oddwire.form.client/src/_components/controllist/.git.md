# Brief ‚Äî Dynamic Form

**Scope:** Feature
**Domain:** oddform
**Status:** Active

---

## What it is

A form runtime that loads a JSON form definition and a sparse instance overlay from a persistent store, renders the merged control tree by dispatching each control on `type`, and writes every edit back to the store by `param`.

---

## Core Loop

```
initialise store (seed JSON first run) ‚Üí getForm(id) + getInstance(id) ‚Üí resolve each control (form ‚äï instance) ‚Üí dispatch by type ‚Üí user edits ‚Üí onChange(value, param, key) ‚Üí setValue merges the overlay ‚Üí re-render + set(instance, id) ‚Üí persist to IndexedDB
```

Inputs: **form definition** (JSON control tree, via `FormContext` repo), **instance overlay** (sparse `param`-keyed patches, via `InstanceContext` repo)
Output: **persisted live instance** ‚Äî survives refresh; reverts to the bundled seed only when the store is cleared.

Store loads once behind a splash; every control edit autosaves the whole instance.

---

## Driving Values

| Value | What drives it | What it does |
|---|---|---|
| `type` | Definition config | Selects the renderer in `ControlItem`; unknown ‚Üí fallback error box |
| `param` | Definition + overlay | Identity that matches an overlay patch to a form control |
| `value` | Definition default ‚Üí overlay ‚Üí user edits | Current control value |
| `key` | Editing control | Which prop of the control an edit targets (defaults to `value`) |
| `hidden` | Definition / overlay | Skips rendering when true (gated per leaf / `ControlBase`) |
| `valueType` | Text control config | On-screen keyboard hint (text/int/decimal/email/phone) |
| `controls` | Radio/Dropdown config | Static option list |

---

## User Loop

| Input | Action | Reward | Risk |
|---|---|---|---|
| Control props + overlay | Fill / toggle / select a control | Value merged into the overlay and persisted | Whole instance written per keystroke; duplicate params collapse to one; missing form id hangs on the loading shell |

---

## Class Responsibility

| Class | Owns |
|---|---|
| `FormContext` / `InstanceContext` | Data repos ‚Äî `getForm(id)` / `getInstance(id)` + `set(instance, id)`, no entity logic |
| `persistentStore` | localforage (IndexedDB) store: seed on first run, resolve by id, persist on set |
| `ContextsProvider` | Initialises the store behind a splash, provides both repos |
| `InstanceEntity` | The live instance ‚Äî value object (`get`/`resolve`/`setValue`) that is also the React component owning the active document + autosave |
| `FormPage` | Loads the definition, hosts `InstanceEntity` |
| `ControlList` | Iterates a control scope ‚Äî no type knowledge |
| `ControlItem` | Resolves one control (form ‚äï overlay) and dispatches by `type` |
| `StripLayout` | Headered tablet-portrait page shell |
| (controls) | Leaf rendering + onChange ‚Äî see `controls.git.md` |
