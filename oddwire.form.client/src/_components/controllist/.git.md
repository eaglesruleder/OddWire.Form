# Brief — Dynamic Form

**Scope:** Feature
**Domain:** oddform
**Status:** Active

---

## What it is

A JSON-driven form runtime: an initialiser seeds two localforage stores (forms, instances) keyed by GUID with derived index tables, a landing page selects a form then an instance (or a new one), and `FormPage` renders the merged control tree (definition ⊕ instance overlay) dispatched by `type`, persisting edits by `param` after an explicit first save.

---

## Core Loop

```
initialise stores (seed JSON first run, build index tables) → LandingPage lists forms → pick form → list its instances or New → route /form/:formId/:instanceId? → FormPage getForm + getInstance → resolve each control (form ⊕ instance) → dispatch by type → user edits → onChange(value, key, subkey) → setValue merges the overlay → hard Save persists + enables autosave → later edits autosave → index re-projected
```

Inputs: **form definition** (GUID-keyed JSON control tree, via `FormStore`), **instance overlay** (sparse `param`-keyed patches, via `InstanceStore`)
Output: **persisted live instance** — survives refresh; a new instance is memory-only until its first Save.

Stores load once behind a splash. `list()` reads only the derived index tables, never the bodies.

---

## Driving Values

| Value | What drives it | What it does |
|---|---|---|
| `type` | Definition config | Selects the renderer in `ControlItem`; unknown → fallback error box |
| `param` | Definition + overlay | Identity matching an overlay patch to a form control; the `onChange` `key` |
| `value` | Definition default → overlay → user edits | Current control value |
| `subkey` | Editing control | Which prop of the control an edit targets (defaults to `value`) |
| `hidden` | Definition / overlay | Skips rendering when true (gated per leaf / `ControlBase`; not honoured on `tab` controls — they bypass `ControlItem`) |
| `valueType` | Text control config | On-screen keyboard hint (text/int/decimal/email/phone) |
| `controls` | Radio/Dropdown config; layout config | Static option list (radio/dropdown) or child control tree (collapsible/tab/popup) |
| control `type === 'tab'` | Definition config | A run of adjacent tabs folds into one tabset; a `tab` first in `form.controls` makes the whole form a root tabset |
| `displayParam` | Form root | Which params project into an instance's index `display` label |
| `version` | Form root | Shown beside the form name on the landing page |
| `dateModified` | Instances: stamped by the store on save. Forms: persisted as published | Shown on the landing lists |

---

## User Loop

| Input | Action | Reward | Risk |
|---|---|---|---|
| Form + instance selection | Pick a form → an existing instance or New | Opens `FormPage` on the merged document | A new instance is discarded if left before the first Save |
| Control edit | Fill / toggle / select a control | Merged into the overlay; autosaved after the first Save | Whole instance + index rewritten per keystroke; duplicate params collapse last-wins |

## Class Responsibility

| Class | Owns |
|---|---|
| `FormContext` / `FormStore` | Forms store (storeName `forms`): GUID body keys + one `index`; `getForm(id)` / `list()` / `saveForm` + `getDisplayParam(id)` for the instance display projection |
| `InstanceContext` / `InstanceStore` | Instances store (storeName `instances`): GUID body keys + one `index`; `getInstance(id)` / `list(formId)` / `save`; projects `display` from `FormStore.getDisplayParam` (FK-style) |
| `ContextsProvider` | Initialises forms then instances behind a splash, provides both stores as context values |
| `InstanceEntity` | The live overlay — value object (`get`/`resolve`/`setValue`) that self-mints its GUID |
| `App` | `BrowserRouter` routes: `/`, `/form/:formId/:instanceId?`, `/settings` |
| `LandingPage` | Form list → instance list (filtered by form) + New; settings gutter link |
| `SettingsPage` | Placeholder shell (DB Manager is MVP+) |
| `FormPage` | Route-driven load; explicit first Save then autosave; header save icon; renders a root tabset when `form.controls[0]` is a `tab` (strays fold into a trailing ⚠ tab) |
| `StripLayout` | Sticky three-cell masthead (left icon · centre title · right icon) over padded `main`; reused as the ControlPopup banner |
| `ControlList` | Iterates a control scope, folding a run of `tab` controls into one inline tabset — otherwise no type knowledge |
| `ControlItem` | Resolves one control (form ⊕ overlay) and dispatches by `type`; passes `instance` to layout controls |
| (controls) | Leaf + layout control rendering + onChange — see `controls.git.md` |
