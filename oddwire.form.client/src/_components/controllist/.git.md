# Brief - Dynamic Form

**Scope:** Feature  
**Domain:** oddform  
**Status:** Active

---

## What It Is

A browser-local JSON form runtime. `ContextsProvider` initialises form, instance, and lookup stores; the landing page selects a form and an existing or new instance; `FormPage` renders the selected definition over a sparse `param`-keyed instance overlay; `ControlItem` dispatches each resolved control by `type`; settings manage lookup tables, bundled form installs, and monster imports.

Definitions, instances, and lookup tables persist through `localforage`. Instance index rows store projected `{ label, value }` fields for display/filter/group/order views. A new instance remains unsaved until the first hard Save; after that, edits are debounced and autosaved.
Forms may declare `export.api` / `export.url` to enable a header API export action that flattens the resolved instance to key/value JSON and posts it to the configured endpoint. Forms may also declare `export.pdf` for client-side PDF export; only controls with a `pdf` placement map print their resolved values at page coordinates. Forms without export config keep the save icon; saved/autosaving instances show a toast on save-icon click.

## Core Loop

```text
initialise stores
-> seed forms / instances / lookup tables when empty
-> LandingPage lists forms
-> expand a form and pick an instance or New
-> route /form/:formId/:instanceId?
-> FormPage loads form + resolves instance
-> ControlList walks the control tree
-> ControlItem resolves definition + instance overlay
-> dispatch by resolved type
-> user edits
-> InstanceEntity.setValue(param, subkey, value)
-> render immediately
-> hard Save persists new instance and enables autosave
-> later edits persist through debounced save
-> optional export action posts flattened resolved values to form export API
-> optional PDF export downloads a client-generated PDF from placed fields
-> instance index projections are re-built from display/group/filter/order params
-> LandingPage renders grouped/filterable/orderable rows from the index
```

Inputs: **form definition** (GUID-keyed JSON control tree), **instance overlay** (sparse patches keyed by `param`), **lookup database** (scope -> table -> rows/schema).  
Output: **saved local instance** plus derived index rows for landing-page display.

## Driving Values

| Value | What drives it | What it does |
|---|---|---|
| `formId` | Route + form body | Loads the definition and lookup scope |
| `instanceId` | Route + instance body | Loads an existing instance; absent means new unsaved instance |
| `type` | Definition, then overlay override | Selects renderer in `ControlItem`; unknown renders `ControlError` |
| `param` | Definition + overlay | Global identity for instance lookup, update, display projection, and React keys |
| `value` | Definition default -> overlay -> edits | Current control value |
| `subkey` | Editing control | Which property is patched, defaulting to `value` |
| `hidden` | Definition or overlay | Skips render; hidden tabs are excluded from tabsets |
| `controls` | Control definition | Static options for radio/dropdown, or children for layout controls |
| looper `value` | Instance overlay | Sparse row-instance array rendered against the looper template |
| `dbOptions` | Radio/dropdown definition | Resolves options from lookup tables, optionally filters/fills from a selected row |
| `displayParam` | Form root | Projects selected instance values into the landing-page title/details; `null` inserts a detail line break |
| `groupParam` | Form root | Groups landing-page instance rows and receives form-page back-link query params |
| `filterParam` | Form root + landing URL params | Builds landing-page filter dropdowns and can preselect values from query params |
| `orderParam` | Form root | Cycles instance ordering on the landing page |
| `projectionLabels` | Form index | Stores labels beside projected values so index views do not need the full form body |
| `version` | Form root | Drives install/update/refresh status in Form Manager |
| `export.api` / `export.url` | Form root | Enables API export from the form header action popup |
| `export.pdf` | Form root | Enables client-side PDF export |
| control `pdf` | Control definition | Maps string page keys to one or more `{ x, y, w?, h? }` print boxes (`x, y` = text bottom-left anchor) |
| control `export.pdf.fontSize` | Control definition | Overrides the default text size for that control's print boxes |
| `export.pdf` | App config (`src/AppSettings.json`) | `fontSize` default for placed text; `showGrid` spacing (px, 0 = off) overlays a coordinate ruler on the export |
| PDF template blob | Package import | Stored under `{formId}/template` and used as the export background |
| `dateModified` | Stores | Instance save stamps it; form save preserves published value; lookup save stamps `lastUpdated` |

## User Loop

| Input | Action | Reward | Risk |
|---|---|---|---|
| Form selection | Expand a form on the landing page or pass `FormID` in the URL | Shows saved instances and New link | Form list is index-derived; stale indexes hide bodies until refreshed |
| New instance | Open `/form/:formId` | Start from definition defaults | Lost if the user leaves before first Save |
| Existing instance | Open `/form/:formId/:instanceId` | Resume saved local data | Duplicate params compress last-wins |
| Control edit | Type/toggle/select | Updates overlay and rerenders immediately | Empty text value drops the param entry |
| Looper add row | Click Add Row | Appends a sparse child instance rendered from the row template | Rows currently cannot be removed or reordered |
| Header actions | Click header icon | No export config: Save icon saves once, then toast-confirms autosave. Export config: popup shows Save/Autosaving, API export, and PDF export when configured. | Export uses current resolved values and does not force a save first |
| DB Manager import | Pick JSON row file and table | Creates or merges lookup rows | Import is JSON-only and collision resolution is key-based |
| Monster import | Filter/load a 5etools source, select rows, set collection/session, import | Builds/updates the global `Monster` lookup table | Remote source shape and token URLs depend on 5etools public paths |

## Class Responsibility

| Class / Area | Owns |
|---|---|
| `App` | Routes `/`, `/form/:formId/:instanceId?`, `/settings`, fallback to `/` |
| `ContextsProvider` | Initialises forms -> instances -> lookup behind a splash and provides stores |
| `FormContext` / `FormStore` | Local form bodies, form index, `displayParam` lookup, save/reindex trigger |
| `InstanceContext` / `InstanceStore` | Local instance bodies, instance index, save/load/list, display projection |
| `LookupContext` / `LookupStore` | Local lookup database, global scope, table CRUD/import, aggregate lookup map |
| `PdfTemplateContext` / `PdfTemplateStore` | Local PDF/image template blobs keyed by form id |
| `InstanceEntity` | Live overlay wrapper, `get`/`resolve`/`setValue`/`setValues`, debounced persistence hooks |
| `LandingPage` | Form list, URL-driven form open, filters/order/group expansion, instance rows, new-instance link |
| `FormPage` | Route load, first save/autosave flow, group-aware back link, error states, root tab layout, header action popup, `DbContext` provider |
| `export/flattenInstance` | Walks the resolved form tree and creates key/value JSON plus PDF placements for export |
| `export/FormPdfExporter` | Parses a form + instance into flattened export data and writes the PDF export |
| `export/PdfWriter` | Wraps `pdf-lib` with the app's text/page writing surface |
| `ControlList` | Iterates a control scope; groups adjacent tabs into one tabset; applies control grid |
| `ControlItem` | Resolves overlay, interpolates label, dispatches by `type`, wires lookup options/fill |
| `lookup/resolveDbOptions` | Normalises `dbOptions`, filters rows, maps rows to options, handles missing/empty option fallback |
| `SettingsPage` | Settings shell, clear-all local data action, DB Manager, Form Manager |
| `DbManager` | Global lookup table picker, schema editor, row editor, JSON import, collision resolution, 5etools monster import entry |
| `FormManager` | Install/update/refresh bundled JSON forms and zip packages, including PDF templates |
| `mods/5etools` | Downloads 5etools bestiary sources, expands supported `_copy`/`_mod` entries, reviews selected monsters, maps them into the global `Monster` lookup table |
| `StripLayout` | Sticky strip masthead and centered page shell |
| `controls/` | Leaf and layout controls; see `controls/.git.md` |
